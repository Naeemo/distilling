# 微信文章抓取方案分析

## 问题
微信文章(mp.weixin.qq.com)有严格的反爬虫机制：
- 检测 headless 浏览器
- 检测自动化工具特征
- 需要环境验证

## 已测试方案

### ❌ Lightpanda
- 被微信检测拦截
- 返回 "环境异常，需要验证"

### ❌ 普通 HTTP 请求
- 微信需要 JavaScript 执行
- 无法获取正文内容

## 可行解决方案

### 方案1: 代理 + 浏览器农场 (成本高)
- 使用住宅代理 IP
- 轮换 User-Agent 和指纹
- 维护成本：~¥500-2000/月

### 方案2: 微信开放平台 API (推荐)
- 申请微信开放平台账号
- 使用官方内容获取接口
- 需要公众号认证
- 稳定合法

### 方案3: 用户手动复制 (MVP 阶段)
- 用户在微信内打开文章
- 复制链接或正文到 InfoDigest
- 最简单的实现
- 零成本

### 方案4: 浏览器扩展
- 开发 Chrome/Edge 扩展
- 用户授权后自动抓取
- 在真实浏览器环境运行
- 绕过检测

### 方案5: 第三方服务
- 使用付费的微信文章解析服务
- 如：Readability API、 Mercury Parser 等
- 成本：~¥0.1-0.5/篇

## 建议 (MVP 阶段)

采用 **方案3 + 方案4** 组合：

1. **短期**: 引导用户手动复制文章内容
2. **中期**: 开发浏览器扩展自动抓取
3. **长期**: 申请微信开放平台 API

这样可以在不增加成本的情况下，先让产品跑起来。

## 实现计划

### 阶段1: 手动粘贴 (本周)
- 修改前端采集界面
- 支持直接粘贴 Markdown/纯文本
- 自动提取链接中的标题

### 阶段2: 浏览器扩展 (下周)
- 开发 Chrome 扩展
- 一键保存当前微信文章
- 调用 InfoDigest API

### 阶段3: 官方 API (后续)
- 申请微信开放平台
- 集成官方内容接口
- 完全自动化采集
