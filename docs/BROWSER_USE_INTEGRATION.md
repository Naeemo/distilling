# Browser Use 集成方案

## 目标
使用 browser-use 替代简单的 HTTP 请求，通过真实浏览器获取微信文章内容。

## 架构设计

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   NestJS API    │────▶│  Browser Use    │────▶│  微信文章页面   │
│                 │     │  (Python服务)   │     │  (真实浏览器)   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │
        ▼                       ▼
┌─────────────────┐     ┌─────────────────┐
│   PostgreSQL    │     │   正文内容      │
│                 │     │   (Markdown)    │
└─────────────────┘     └─────────────────┘
```

## 实现方案

### 方案1: Python 微服务 (推荐)
- 独立 Python 服务运行 browser-use
- NestJS 通过 HTTP API 调用
- 异步处理，支持并发

### 方案2: 直接集成
- 使用 Node.js 的 Playwright
- 直接嵌入 NestJS
- 更简单，但可能阻塞主线程

## 选择: 方案2 (Node.js Playwright)

原因:
1. 与现有技术栈一致 (Node.js)
2. 无需额外 Python 服务
3. 部署更简单

## 实施步骤

1. 安装 Playwright
2. 创建 BrowserService
3. 替换现有内容采集逻辑
4. 添加微信文章专用处理
