# 首次工作流

这一页介绍第一次使用 InfoDigest 时最简单的一条路径。

## 1. 在 Web 应用中登录

从 Web 应用开始。使用 Google、GitHub 或邮箱 magic link 完成登录后，仪表盘就是你查看已保存内容和添加新条目的主要位置。

## 2. 添加第一个条目

使用仪表盘上的 Quick Collect 输入框。

它支持：

- 普通文章 URL
- 微信分享链接
- 包含微信 URL 的分享文本
- 纯文本或 Markdown

如果你粘贴的是文本而不是 URL，InfoDigest 会把它存成类似笔记的内容条目。

保存完成后，系统会自动开始生成摘要；你不需要再额外打开条目手动触发一次。

## 3. 打开已保存条目

创建完成后，在仪表盘列表里打开它。

你通常会看到：

- 保存后的标题
- 使用 URL 时的来源信息
- 例如未读或阅读中这样的阅读状态
- 自动生成中的摘要，或稍后可用的摘要内容

## 4. 主动阅读

阅读条目时，可以在阅读器中：

- 高亮重要段落
- 添加简短笔记
- 给条目打标签，方便后续筛选
- 推进阅读状态

## 5. 之后再次回来

当你积累起一个资料库后，可以使用：

- [阅读与复习](/zh/workflows/read-and-review) 回看重要材料
- [知识图谱](/zh/workflows/knowledge-graph) 理解条目之间的联系

## 什么时候切换到其他收集方式

- 当微信公众号文章已经在浏览器中打开时，使用 [浏览器扩展](/zh/platforms/browser-extension)。
- 当你想要一条更轻量的移动端采集路径时，使用 [移动端快捷指令](/zh/platforms/mobile-shortcut)。
