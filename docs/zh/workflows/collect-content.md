# 收集内容

InfoDigest 支持多种收集路径，它们分别适用于不同场景。

## 选择合适的路径

| 场景 | 最佳路径 |
| --- | --- |
| 普通文章或博客 | Web 应用中的 Quick Collect |
| 微信文章已在 Chrome 中打开 | 浏览器扩展 |
| 复制来的笔记、转录稿或 Markdown | Web 应用中的文本模式 Quick Collect |
| 以分享为主的移动端采集 | iOS 快捷指令，或粘贴到 Web 应用 |

## Web 应用 Quick Collect

Quick Collect 是大多数用户的默认路径。

你可以粘贴：

- 标准 URL
- 微信文章 URL
- 一段微信分享文本
- 纯文本
- Markdown

InfoDigest 会识别输入内容，并决定是抓取网页还是直接保存文本。

当你提交 URL、微信文章或粘贴文本后，条目保存完成就会自动开始生成摘要，不需要再进入阅读器手动点一次“生成摘要”。

如果某篇内容已经被其他用户抓取并完成前段消化，系统会优先复用已有结果，让新的提交更快进入可阅读状态。

## 浏览器扩展

当微信文章已经在浏览器中打开时，使用扩展。

它的价值在于：

- 运行在真实页面上下文中
- 可以直接读取标签页中的文章内容
- 减少把链接再复制回 Web 应用的摩擦

在你已经登录 Web 应用后，扩展通常效果最好。

具体流程请阅读 [浏览器扩展](/zh/platforms/browser-extension)。

## iOS 快捷指令流程

仓库中包含一份用于轻量移动端采集的 iOS 快捷指令参考配置。

以下情况适合使用它：

- 你想从 iOS 传递链接或分享内容
- 你能够接受基于快捷指令加单独 Integration Token 的设置方式
- 你不需要独立的原生 App

详细说明见 [移动端快捷指令](/zh/platforms/mobile-shortcut)。

## 收集之后会发生什么

条目创建后，它会出现在你的资料库中，并进入产品后续流程：

- 阅读
- 高亮
- 打标签
- 自动摘要
- 复习
- 图谱探索

如果你想回看每次是什么时候加进来的、是否重复提交、当前处理到了哪一步，可以在 Web 应用的“添加记录”页面查看时间轴。

## 实用建议

- 想要完整文章提取时，优先使用 URL。
- 想按原样保留笔记时，使用粘贴文本。
- 处理微信内容时，如果仅靠复制链接提取不稳定，优先使用扩展。
