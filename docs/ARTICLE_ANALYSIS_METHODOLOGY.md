# 知萃文章分析方法论

**核心目标**：建立一套系统性的文章解析框架，能够识别文章质量、提取核心价值、发现逻辑问题。

---

## 一、分析维度总览

```
┌─────────────────────────────────────────────────────────────┐
│                    文章质量评估金字塔                          │
├─────────────────────────────────────────────────────────────┤
│  Level 4: 价值判断  →  这篇文章值得读吗？                      │
│  Level 3: 逻辑检验  →  论证是否成立？                          │
│  Level 2: 信息萃取  →  作者到底说了什么？                      │
│  Level 1: 元数据扫描 → 这是什么类型的文章？                    │
└─────────────────────────────────────────────────────────────┘
```

---

## 二、Level 1: 元数据扫描（30秒内完成）

### 2.1 文章类型识别

| 类型 | 特征 | 预期价值 |
|------|------|----------|
| **资讯报道** | 时间地点人物事件，5W1H | 时效性信息 |
| **观点评论** | 主观判断词多，"应该""必须" | 视角参考价值 |
| **知识科普** | 概念解释，逻辑递进 | 学习价值 |
| **科研论文** | 方法论、数据、引用 | 深度研究价值 |
| **营销软文** | 痛点→产品→转化 | 警惕 |
| **情绪宣泄** | 感叹号、情绪词密集 | 极低 |

### 2.2 信源质量评估

**可信指标** (+)
- 可验证的引用来源
- 作者署名 + 专业背景
- 发布机构有编辑审核流程
- 数据有明确出处

**可疑指标** (-)
- 匿名/化名作者
- "据说""有网友称"等模糊信源
- 利益相关方（产品推广文）
- 情绪化标题党

---

## 三、Level 2: 信息萃取（2分钟内完成）

### 3.1 核心要素提取

**必提取字段**：
```
论题(Question): 作者试图回答什么问题？
结论(Claim): 作者的核心观点是什么？
理由(Reason): 支持结论的主要论据？
证据(Evidence): 数据、案例、引用？
```

**提取技巧**：
- 首段通常包含论题
- 寻找指示词："因此""所以""证明"
- 段首句往往是分论点

### 3.2 结构分析

**论证结构图**：
```
         [结论]
        /   |   \
    [理由1] [理由2] [理由3]
      |       |       |
   [证据1]  [证据2] [证据3]
```

**常见结构模式**：
- **金字塔型**：结论→分层理由→底层证据（学术论文）
- **故事型**：情境→冲突→解决方案（商业文章）
- **列表型**：观点1→观点2→观点3（清单文）

## 四、Level 3: 逻辑检验（核心环节）

### 4.1 逻辑谬误检测清单

#### A. 人身相关谬误
| 谬误 | 特征 | 示例 |
|------|------|------|
| **人身攻击** | 攻击作者而非论点 | "他学历低，观点不可信" |
| **诉诸权威** | 盲目引用权威 | "专家说...所以是对的" |
| **诉诸公众** | "大家都信" | "朋友圈都转发了" |

#### B. 论据相关谬误
| 谬误 | 特征 | 示例 |
|------|------|------|
| **稻草人** | 曲解对方观点后攻击 | "他说要环保，就是要让大家都穷" |
| **虚假因果** | 相关≠因果 | "吃冰淇淋多的时候溺水多，所以冰淇淋导致溺水" |
| **以偏概全** | 小样本推大结论 | "我认识的程序员都秃头" |
| **滑坡谬误** | A→B→C无限推演 | "如果不加班，公司就会倒闭" |

#### C. 假设相关谬误
| 谬误 | 特征 | 示例 |
|------|------|------|
| **非黑即白** | 忽略中间地带 | "不支持就是反对" |
| **循环论证** | 结论当理由 | "因为A，所以A" |
| **错误类比** | 本质不同的类比 | "国家就像家庭，不能入不敷出" |

### 4.2 证据强度评估

**证据等级（从高到低）**：
1. **系统综述/Meta分析** > 随机对照试验 > 观察性研究 > 个案报告
2. **原始数据** > 二手引用
3. **多方验证** > 单一来源
4. **近期数据** > 过时数据

**红旗信号** 🚩：
- 数据没有来源标注
- "研究表明"但不提具体研究
- 统计口径不明确（平均值？中位数？）
- 图表坐标轴被裁剪

## 五、Level 4: 价值判断（最终输出）

### 5.1 文章质量评级

| 评级 | 标准 | 处理建议 |
|------|------|----------|
| **A级** | 逻辑严密+证据充分+观点新颖 | 精读+做笔记+加入复习 |
| **B级** | 逻辑基本通顺，但有瑕疵 | 提取核心观点即可 |
| **C级** | 逻辑问题多，或证据薄弱 | 仅提取关键信息 |
| **D级** | 明显谬误/营销软文/情绪文 | 直接丢弃 |

### 5.2 决策树

```
                    开始分析
                       │
            ┌──────────┴──────────┐
            ▼                      ▼
      信源可信？              信源可疑？
            │                      │
     是 ┌───┘                      └───┐ 否
        ▼                              ▼
   提取核心论点                    标记"需谨慎"
        │                              │
        ▼                              ▼
   逻辑检验通过？                   内容有价值？
        │                              │
   是 ┌─┴─┐ 否                   是 ┌─┴─┐ 否
      ▼   ▼                        ▼     ▼
    A/B级 C级                    提取要点  丢弃
```

## 六、心理学视角：认知偏见的对抗

### 6.1 读者端偏见（我们自身）

| 偏见 | 表现 | 对抗方法 |
|------|------|----------|
| **确认偏误** | 只看支持自己观点的信息 | 主动寻找反面观点 |
| **达克效应** | 高估自己的理解能力 | 复述测试：能否用自己的话解释？ |
| **锚定效应** | 被首因信息过度影响 | 延迟判断，先看完整内容 |
| **可得性启发** | 容易想起的例子被认为更常见 | 追问：这是特例还是规律？ |

### 6.2 作者端偏见（文章本身）

**识别操纵技巧**：
- **情感词汇**："震惊！""太可怕了！"
- **虚假紧迫感**："错过后悔一辈子"
- **社会认同压力**："聪明人都这样做"
- **稀缺性暗示**："最后机会"

## 七、新闻学视角：5W1H + 交叉验证

### 7.1 基础要素检查

```
Who    → 谁写的？谁受益？谁受害？
What   → 发生了什么？具体是什么问题？
When   → 什么时候？时效性如何？
Where  → 哪里发生的？地域是否有代表性？
Why    → 为什么会这样？还有其他原因吗？
How    → 如何发生的？过程是否清晰？
```

### 7.2 交叉验证原则

**单一信源 = 不可信**

验证方法：
1. 同一事件是否有其他媒体报道？
2. 原始出处能否找到？
3. 数据和结论是否匹配？

### 8.1 分析流程

```python
def analyze_article(text: str, metadata: dict) -> AnalysisResult:
    # Step 1: 元数据扫描
    article_type = classify_type(text)
    source_credibility = assess_source(metadata)
    
    # Step 2: 信息萃取
    question = extract_question(text)
    claim = extract_claim(text)
    reasons = extract_reasons(text)
    evidence = extract_evidence(text)
    
    # Step 3: 逻辑检验
    fallacies = detect_fallacies(text)
    evidence_quality = assess_evidence_strength(evidence)
    
    # Step 4: 价值判断
    quality_score = calculate_quality_score(
        source_credibility,
        logical_soundness,
        evidence_quality
    )
    
    return AnalysisResult(
        quality_rating=quality_score,
        key_takeaways=extract_key_takeaways(text),
        logical_issues=fallacies,
        reading_recommendation=recommend_action(quality_score)
    )
```

## 九、实践案例

### 案例1：识别水文

**文章特征**：
- 标题："震惊！这个习惯让90%的人癌症"
- 内容：没有具体研究引用，只有"有研究表明"
- 结构：恐吓→产品推荐→购买链接

**分析结果**：
- 质量评级：D
- 主要问题：虚假因果 + 营销目的
- 建议：直接丢弃

### 案例2：识别干货

**文章特征**：
- 标题："深度：新能源汽车电池技术路线比较"
- 内容：数据图表 + 技术原理 + 优缺点对比
- 引用：标注具体研究报告和企业财报

**分析结果**：
- 质量评级：A
- 价值点：技术对比框架、数据支撑
- 建议：精读+做笔记
